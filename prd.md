# [PRD] d-maker.kr 뉴스 자동/수동 수집 및 업로드 시스템 구축

## 1. 프로젝트 개요
**목표:** 외부 뉴스 제공 API(NewsData.io 및 뉴스와이어)와 연동하여 뉴스를 주기적(Cron) 또는 수동으로 가져오고, 수집된 데이터를 자사 뉴스 사이트(d-maker.kr)에 자동 또는 수동으로 퍼블리싱하는 통합 관리 시스템을 구축한다 [1].

## 2. 연동 대상 API 환경
본 시스템은 두 가지 특성이 다른 뉴스 API를 연동합니다 [1].

### 2.1. NewsData.io API (글로벌 뉴스 및 과거 데이터 검색)
* **형태:** JSON 기반의 REST API [2].
* **특징:** 전 세계 97,876개 이상의 뉴스 소스에서 키워드, 국가, 언어, 카테고리 등을 기반으로 실시간 및 과거 뉴스를 수집한다 [2].

### 2.2. 뉴스와이어 (제휴사 보도자료 전송 API)
* **형태:** JSON 기반의 REST API (`/api/v1/request` 호출 후 `/api/v1/send` 호출하는 2-Step 방식) [2].
* **특징:** 국내 기업 보도자료 수신을 목적으로 하며, HMAC 인증 방식(`X-HMAC`, `X-Timestamp`)을 사용한다 [2].

## 3. 주요 기능 요구사항

### 3.1. 뉴스 수집(Fetcher) 모듈
* **자동 수집 (스케줄러):**
  * **뉴스와이어:** 5~10분 간격으로 API 서버에 접속해 스케줄링 방식으로 보도자료를 수신하며, 한 번에 받을 수 있는 보도자료는 최대 100건이다 [2].
  * **NewsData.io:** 설정된 키워드(`q`), 카테고리(`category`), 언어(`language`) 등의 파라미터를 활용해 최신 뉴스(`latest`)를 주기적으로 호출한다 [2].
* **수동 수집 (어드민 기능):**
  * 관리자 페이지에서 특정 키워드나 발행일(`from_date`, `to_date` 등)을 입력하여, 원할 때 수동으로 데이터를 즉시 불러오는 기능을 제공한다 [2].
  * 뉴스와이어의 경우 특정 장애 발생 시 발급받았던 `send_id`를 사용하여 재수신을 수동으로 요청할 수 있어야 한다 [2].

### 3.2. 데이터 전처리 및 저장(Processor) 모듈
API로 가져온 데이터를 d-maker.kr의 DB 스키마에 맞게 매핑(Mapping)하고 가공한다 [3].

* **뉴스와이어 데이터 처리 규칙 (필수 준수):**
  * **액션(Action) 분기:** 응답 데이터의 `action` 값에 따라 반드시 순서대로 처리한다 [3]. 
    * `insert`: 신규 등록 처리 [3].
    * `update`: 기존 데이터 수정 (DB에 없으면 신규 `insert`로 처리) [3].
    * `delete`: 데이터 삭제 (DB에 없으면 무시) [3].
  * **엠바고(보도시간) 체크:** `press_time`(보도자료 발표시각) 값을 체크하여, 현재 시간보다 미래인 경우에는 사이트에 노출되지 않도록 처리한다 [3].
  * **이미지 다운로드 및 호스팅:** 제공된 이미지 URL(`photo.url`)은 **외부 URL 직접 링크가 금지**되어 있으므로, 반드시 d-maker.kr의 자체 서버로 이미지를 다운로드 및 저장한 뒤 자체 호스팅으로 서비스해야 한다 [3].
* **NewsData.io 데이터 처리 규칙 (필수 준수):**
  * **저작권 보호:** 이미지와 비디오는 저작권 문제 방지를 위해 원본이 아닌 '썸네일 크기'로 렌더링하도록 처리해야 한다 [3].
  * **데이터 중복 방지:** API에서 제공하는 `article_id`를 고유 식별자로 사용하여 중복 저장을 방지한다 [3]. 기사 내용 게재 시 원본 출처(Publisher) 크레딧을 명시해야 한다 [3].

### 3.3. 뉴스 업로드 및 발행(Publisher) 모듈 (d-maker.kr 연동)
* **자동 업로드:** 수집 및 전처리가 완료된 데이터는 `press_time` 등의 제약 조건에 어긋나지 않는 한 d-maker.kr DB에 즉시 발행 상태로 Insert 된다 [4].
* **수동 업로드 (검수 프로세스):**
  * 데이터를 수집하되 상태값을 '임시저장(Draft)'으로 유지한다 [4].
  * 관리자가 백오피스에서 제목, 본문, 이미지를 검토 및 수정한 후 '발행' 버튼을 클릭하여 수동으로 사이트에 업로드할 수 있는 UI를 제공한다 [4].
* **콜백(Callback) 시스템 (뉴스와이어):**
  * 뉴스와이어 보도자료가 d-maker.kr에 성공적으로 발행되어 공개 URL이 생성되면, `/api/v1/callback/{pid}` API를 호출하여 보도자료가 실제로 사용된 외부 공개 URL을 뉴스와이어 측에 전달(POST)한다 (내부망 주소 제외) [4].

## 4. 시스템 인증 및 에러 핸들링 (비기능 요구사항)
* **보안 및 인증:**
  * 뉴스와이어 API 호출 시 Header에 발급받은 API Key와 sha256을 이용해 생성한 `X-HMAC`을 포함해야 한다 [4].
  * `X-Timestamp`는 UTC 기준 경과 시간(Millisecond)을 사용하며, d-maker.kr 서버와 API 서버 간 시간 차가 5분 이상 나지 않도록 서버 시간(NTP)을 철저히 동기화해야 한다 [4].
* **에러 핸들링:** 각 API 호출 시 HTTP 상태 코드 (예: 200 성공, 401 인증 실패, 500 서버 내부 오류 등)에 대한 로깅 및 알림 시스템을 구축한다 [4].
